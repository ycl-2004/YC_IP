# Explanation Variation：解释类场景原型

用于“解释 X / 什么是 X / 用 YC 讲清一个概念”的 Sample / Standard 图片。目标是保留 YC 的红发眼镜、白底手绘、中文短标注和因果可读性，同时避免每张都变成同一台机器、箱子或曲柄装置。

## 先选场景原型，再写 prompt

解释类图片不要默认等于“机器输入输出”。先根据主题选择一个场景原型；只有确实需要展示内部转换、过滤、压缩、分类、预测时，才使用装置原型。

| 场景原型 | 适合主题 | 画面语言 | 首选参考 |
| --- | --- | --- | --- |
| 工作台装置 | 分类、预测、编码、过滤、压缩、模型机制 | YC 操作一件有入口、内部痕迹、出口的低科技装置；内部有网、筛、轨道、刻度或模板 | `05-concept-explainer.png` / `07-noise-filter.png` |
| 桥 / 路径 | 信任、迁移、连接、从 A 到 B、关系建立 | YC 铺桥、搬证据、搭木板、点亮路标；重点是“靠什么跨过去” | `02-trust-bridge.png` |
| 分拣 / 收纳 | 优先级、分类、定位、筛选、用途判断 | YC 把素材放入篮子、抽屉、托盘、标签区；重点是“为什么归到这里” | `03-purpose-sort.png` |
| 一源多出 | 复用、内容改写、多平台、多格式输出 | YC 从一个源头拉出多条表达路径；重点是“同一原料如何变成多个成品” | `04-one-source-many-outputs.png` |
| 循环 / 迭代 | 习惯、反馈、实验、优化、复盘、持续改进 | YC 在试做、反馈、调整、再跑之间形成闭环；允许同一个 YC 以小 vignettes 出现 | `08-minimum-loop.png` |
| 接力 / 校准 | 人机协作、审核、决策、取舍、发布前把关 | AI/工具给初稿，YC 校准、取舍、盖章或放行；重点是“人做最后判断” | `09-human-ai-relay.png` |
| 桌面 / 专注角落 | 写作、coding、深度工作、创作状态、思考 | YC 在一个克制工作台中操作笔记本、灯、笔记；重点是“状态和动作” | `11-deep-work-corner.png` |
| 保护 / 边界 | 专注、防干扰、取舍、信息过载、注意力管理 | YC 用伞、屏障、门、罩子保护核心工作；外部干扰以小气泡/便签出现 | `12-protect-focus.png` |
| 倾听 / 采样 | 用户研究、需求、共情、社群反馈、内容选题 | YC 记录多个声音，把“困惑/喜欢/需要”收进笔记；重点是“听见什么” | `13-audience-listening.png` |
| 情绪转换 | 音乐、情绪、表达、灵感、创作 | YC 把心情变成节奏、旋律、歌词或作品；重点是“情绪如何被转译” | `10-emotion-to-melody.png` |
| 小胜利 / 成长 | 进步、鼓励、阶段成果、完成感 | YC 举起成果、贴星星、浇小苗；重点是“小结果如何被看见” | `14-tiny-win.png` |

## 选择规则

1. **先读主题动词**：分类/筛选/预测才偏装置；连接/证明偏桥；复盘/优化偏循环；协作/审核偏接力；专注/写作偏桌面；干扰/边界偏保护。
2. **同一类连续生成时换原型**：如果最近一张已经是机器装置，下一张解释图优先从桥、路径、循环、桌面、保护、倾听里选，除非主题强制需要内部机制。
3. **保留因果，不保留固定布局**：每个原型仍要回答“为什么会得到这个结果”，但不一定必须左输入、中机器、右输出。
4. **场景物件要有职责**：桥板、伞、笔记、反馈箱、抽屉、路标、乐器、灯泡都必须承载解释功能，不能只是装饰。
5. **参考图只锁语言**：使用对应 example 锁角色、线条、白底、密度和标注方式；不要复制原图构图或故事。

## Prompt 填写要求

在 Sample prompt 中增加并填写：

- `Scene archetype`: 从上表选一个。
- `Why this archetype`: 一句话说明为什么它比机器/箱子更适合。
- `YC's core action`: YC 亲手做的动作。
- `Causal proof`: 画面中证明概念成立的具体物件或结果。

如果选择的不是工作台装置，不要强行写“入口/内部/出口”。改写成对应原型的因果：

- 桥：起点 → YC 铺桥/补证据 → 能跨过去的原因 → 到达结果
- 循环：试做 → 反馈 → 调整 → 再跑后变好
- 保护：干扰靠近 → YC 建边界 → 核心工作保持清晰 → 产出
- 倾听：多个声音 → YC 记录/归纳 → 发现共同点 → 做出回应
- 情绪转换：心情 → YC 采样/调节 → 形成节奏/旋律/表达 → 作品

## 失败信号

- 连续解释图都出现同款木箱、曲柄、传送带、标签卡片。
- 主题是信任、情绪、专注、反馈、边界、倾听，却仍被硬塞进机器。
- YC 只是站在装置旁边，动作不改变结果。
- 画面只有“输入 → 处理 → 输出”三个泛化节点，看不出具体情境。
