# YC 视觉风格 DNA

## 一句话

YC 的视觉是**三档密度**，默认走当前 samples 的中等密度：

- **Sample / Standard（默认，最常用）**：对齐 `assets/examples/simple/current-showcase/` 的成品风格。纯白底、温暖手绘 chibi、YC 参与核心动作、中文短标注、中等信息密度。
- **Minimal / Sticker（仅显式点名）**：纯白底上一张水彩铅笔手绘小贴纸。像把 YC 从笔记本边角剪下来贴在白纸上——轻、干净、留白多、几乎没文字。
- **Rich 丰富（仅显式点名）**：完整海报 / IP 设定页 / 品牌全景。只有用户明确说 Rich、完整海报、整页 IP 介绍或品牌全景时才启用。

> ⚠️ **防跑偏第一定律**：模型默认先验是"全彩 anime + 满版拼贴"=海报。不主动压制（纯白背景、手绘线条、YC 做核心动作、中文短标注、带参考图），三档都会滑向海报。**怀疑时回到 Sample / Standard；Minimal 和 Rich 都必须显式点名。**

不是商业海报，不是 PPT，是有温度的创作者内容。

---

## Sample / Standard 默认模式

目标是对齐当前成品样例，而不是生成最轻贴纸或完整海报。基准：`assets/examples/simple/current-showcase/`。

- **背景**：纯白 #FFFFFF，不要整张奶油白/米白纸底；只允许角色脚下、物件下方有很轻的接触阴影
- **画风**：温暖手绘 chibi 插画，保留铅笔/草图质感，但完成度高于 Minimal
- **方法场景化**：把抽象方法画成压缩机、桥、分拣机、传送带、过滤器、输出装置等；YC 要在其中推、压、铺、搬、分拣或试做
- **大概念隐喻化**：遇到“什么是 X”时，不画百科页。先读 `concept-director.md`；用一个中心装置串起具体例子、YC 操作、可见中间结构、新输入和具体判断
- **签名道具**：按场景选 1-3 个（见下表）。工作/创意场景可带 MacBook（♥贴纸）和笑脸咖啡杯 ☺
- **文字**：中文短标注为主，每处 2-8 个字；可以有少量箭头/路径，不写长段解释
- **装饰**：少量心形 ♥ / 星星 ✦（克制）
- **连接**：箭头或流动路径串联，**不要**重型面板边框和 section header（那会变满版 editorial）
- **知识解释限制**：不要同时塞定义、分类、流程、真实案例和总结；一张图只讲一层机制
- **禁止贴纸感拼贴**：不要把 YC 放在带白色矩形底的截图/贴纸里；人物必须自然融入场景，脚下有轻阴影或接触关系
- **必带参考图**：先按 `reference-routing.md` 扫描并选择一张主题匹配的真实 asset；其他 Sample 只选一张同模式、同构图密度的 primary reference
- **机制图 fallback**：`assets/examples/simple/current-showcase/07-noise-filter.png` 仅适用于没有更贴切 topic asset 的单 YC 连续机制场景

## Minimal / Sticker 贴图模式（仅显式点名）

只有用户明确说 Minimal / Sticker / 贴图纸 / 贴纸 / 极简 / 最小贴纸 / minimum 时才启用。目标是**一张小贴纸**，不是默认文章配图、更不是海报。基准：白底上单个 YC 的水彩铅笔手绘小图（`assets/examples/sticker/00-reference-sheet-watercolor.png`）。

- **背景**：纯白 #FFFFFF（不是奶油白、不是纸纹、不是渐变阴影）
- **画风**：水彩铅笔/彩色铅笔轻涂——线松、有轻微抖动、上色淡、透气。不是精致全彩 anime，也不是纯黑线稿
- **YC**：保留红发可识别，但松手绘轻渲染，不是高光厚涂成品
- **主体**：一个 YC + 最多一个简单物件/动作。不拼贴、不分格、不串联
- **颜色**：红发为主色锚点，配淡暖色点缀（米黄/软蓝/软粉），整体低饱和
- **文字**：0-3 个中文短词（能不写就不写）；无英文、无中英混排、无 section header
- **装饰**：零或极少。无满天心形/星星、无便利贴墙、无大"YC"logo、无道具围一圈
- **道具**：最多 1 个且和这张图的意思直接相关——**不为"signature"硬塞 MacBook/笑脸杯**
- **留白**：至少 50%
- **必带参考图**：00 画风锁（长相不稳时加 character-reference 九宫格）

## Rich 丰富模式（仅显式点名）

Rich 是完整成品页，不是默认文章配图。只有用户明确说“Rich / 完整海报 / IP 设定页 / 品牌全景 / 整页成品”时使用。

- **背景**：奶油白 / 轻纸感，允许更丰富的页面结构
- **画风**：精致 editorial chibi 插画，仍保持手绘温度，不做商业 KV 厚重质感
- **主体**：YC 可以出现为主视觉，也可以在多个小场景中重复出现
- **结构**：多区块、多面板、角色设定、内容支柱、品牌生态、工具/生活模式等
- **文字**：允许中英混排和多标注，但每个区块仍短句化，不塞长段正文
- **用途**：IP 自我介绍、品牌全景、内容生态地图、完整海报、个人设定页
- **必带参考图**：`assets/examples/rich/` 中最接近的例图

Rich 的判断规则：用户没明确点名，就不要用。文章封面、正文配图、社媒单图默认仍走 Sample / Standard。

### 色板
| 颜色 | 用途 |
|------|------|
| 深红/酒红 | YC 头发、品牌强调色 |
| 牛仔蓝 | 流程箭头、技术类标注 |
| 黑色 | 线稿、主要文字 |
| 白色 | Minimal / Sample 背景底色 |
| 软粉色 | 心形装饰、情感类标注 |
| 灰色 | 次要文字、辅助线 |

蓝=技术/系统，粉=情感/生活，红=YC signature。

### 签名道具清单（Sample 用，按场景选 1-3）
| 道具 | 场景 |
|------|------|
| MacBook（♥贴纸） | 工作、编程、写作、创作 |
| 笑脸咖啡杯 ☺ | 工作/创意场景 |
| MIDI 键盘 / 头戴耳机 | 音乐/专注 |
| 螺旋/硬封面笔记本 | 想法、规划、学习 |
| 铅笔杯 | 创作 |
| 小植物 | 生活/咖啡馆 |
| 书籍叠放 | 学习/成长 |
| 背包 | 出行/留学/户外 |

## 绝对不要（三模式通用）
- YC 缺红发或眼镜 / 纯黑框眼镜（应是透明细框）
- 过度商业插画感（精修、过亮、饱和过高）
- 正式流程图 / PPT 信息图
- 深色/黑色背景、霓虹发光、赛博机器、照片、显示器 UI
- 顶部标题段、解释段落、编号步骤、底部流程条或总结条
- Broad Concept 出现多个 YC、多个面板或卡片网格
- 线框方法图 + 旁边贴 YC 角色
- 角色旁边有白色矩形边框或截图残留底色
- 过度可爱幼稚（温暖但成熟）
- **没被点名就启用 Minimal**。Minimal 是公开模式，但不是默认模式
- **没被点名就启用 Rich**。Rich 是公开模式，但不是默认模式
