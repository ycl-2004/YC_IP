# YC 个人 IP 配图

## 核心定位

用 YC 的视觉 IP 为任何内容生成配图。主题不限（AI/成长/音乐/设计/生活是常见场景，但讲数学、科学、历史、任何知识文章都适用）——只要是"YC 来给你解释"就行。

YC 角色：深红凌乱短发 + 圆框透明眼镜 + chibi 比例。**红发和眼镜是识别命脉，必须有。**

## 两种模式（默认 Sticker）

只有一种默认密度 = 轻（Sticker）。**靠换"画什么内容"适配场景，不靠加密度。怀疑时永远选 Sticker。**

| | Sticker 贴图（默认，90% 场景） | Simple 普通（仅多步流程） |
|---|---|---|
| 画风 | **水彩铅笔轻涂松手绘**（非全彩 anime，非黑线稿） | 温暖 chibi 插画 |
| 背景 | **纯白 #FFFFFF**，无纹理 | 奶油白，轻纸感 |
| 主体 | 单角色 + 一个动作/物件 | 3-6 步场景串联，箭头连接 |
| 文字 | 0-3 个中文短词（能不写就不写） | 每步 1-2 个短标注 |
| 信息密度 | 极低，巨量留白 | 中：一个流程讲清楚 |
| 适合 | 文章配图、封面/标题、社媒单图、一个概念/情绪/判断 | 流程、步骤、使用方法、before/after |

**怎么选**：
- 一件事（一个概念/观点/情绪，或一个角色在做某事，含封面/标题）→ **Sticker**
- 多步过程（非得分 3-6 步 / 左右对比才说得清）→ **Simple**

> ⚠️ 模型默认会跑成"全彩 anime + 满版拼贴"= 海报。Sticker/Simple 都必须主动压制：白底、松手绘、删道具装饰、删英文、**带参考图**。

## 比例

默认：**文章配图 `16:9`，社媒单图 `1:1`**。其余（`9:16` `4:3` `3:2` `2:3` `5:4` `21:9`）只在用户明确点名时用，不要每次都让用户选。

## 先读这些参考

**默认 Sticker 路径只需两个文件**（角色关键词已并进模板，别一次塞满上下文）：
- `references/prompt-template.md`：生图提示词模板（含两模式角色关键词、约束、参考图指令）
- `references/qa-checklist.md`：生成后检查

按需再读：
- `references/style-dna.md`：两模式画风细节、色板、禁忌（拿不准画风时）
- `references/yc-character.md`：服装/道具库、表情库（要换装或定姿势时）
- `references/yc-brand-identity.md`：5大内容支柱、性格、价值观（讲品牌/自我介绍时）

> `references/composition-patterns.md` 和 Rich 满版模式仅在用户明确要"整页 IP 介绍/品牌全景设定页"这类极少数场景时才读，不进默认流程。

## 工作流

### 1. 消化内容

读用户给的内容，提炼核心观点和"认知锚点"（核心判断/转折/对比/闭环）。**不要每段都配图**——只给锚点配，够用就好，别做成画册。文章默认 3-6 张；单条社媒 1-2 张。

### 2. 先出配图策略（用户说"分析/shot list/帮我想"时）

每张写：放在哪段后、主题+核心意思、用哪个模式、YC 在做什么、建议比例、建议元素和标注词。简短即可。

### 3. 单张生成（必须带参考图）

用户说"生成/出图/帮我做"时，用 `image_gen` **每张单独生成**，不要拼一起。纯文字 prompt 会跑成海报，每张都附参考图：

- Sticker → `assets/examples/sticker/`（同时锁画风+角色长相）
- Simple → `assets/examples/simple/`
- 角色长相不稳时额外加 `assets/examples/character-reference/`

每张提示词必含：明确比例 + YC 关键词（红发 + 透明眼镜 + chibi）+ 模式画风要点。第一次仍偏属正常，带参考图重生一次即对。

`assets/examples/` 只校准画风+角色，**不照抄构图**——每次从当前内容重新想画面。

### 4. 检查与迭代

过 `qa-checklist.md`。常见问题：
- YC 没红发/眼镜 → 角色失败，重生成
- Sticker：背景非纯白、画太精致全彩、出现道具/装饰/英文、太满 → 重生压回贴纸感
- Simple：背景该是奶油白不是纯白、布局太空 → 补回
- 拿不准回看对应模式例图对齐

### 5. 保存

```
assets/<content-slug>/01-topic-name.png
```

## 输出口径

策略输出简洁。交付含：生成张数、每张用途、保存路径、哪张最稳。不解释风格理论，让图说话。
