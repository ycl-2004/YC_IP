# YC 个人 IP 配图

## 核心定位

用 YC 的视觉 IP 为任何内容生成配图。

内容主题不限：YC 的5大内容支柱（AI/成长/音乐/设计/生活）是最常见场景，但也可以讲数学、解释科学概念、介绍历史、配任何知识文章——只要是"YC 来给你解释"或"YC 风格呈现"就适用。

## 两种输出模式

| | Rich 丰富模式 | Simple 极简模式 |
|---|---|---|
| **风格** | 温暖手账 editorial，多面板 | 极简图说，一张图一个隐喻 |
| **文字量** | 中英混排，多 section header，5-10 个标注 | 极少文字，最多 2-4 个短标注 |
| **布局** | 多卡片/面板/分区 | 单场景，大量留白 |
| **信息密度** | 高：一张图能装完整知识体系 | 低：一张图只讲一个核心画面 |
| **适合** | IP 介绍、品牌生态、深度系统、知识全景 | 文章配图、社媒单张、概念隐喻、情绪/场景 |

**如何选择**：
- 用户说"简洁/极简/simple/干净/少字"→ Simple 模式
- 用户说"丰富/详细/rich/完整/全面"→ Rich 模式
- 未指定 → 默认 Simple 模式（文字少更安全）

## 支持的图片比例

| 比例 | 用途 |
|------|------|
| `16:9` | 文章配图、博客 banner、B站封面、横版知识帖 |
| `1:1` | Instagram、小红书方图、通用社媒 |
| `9:16` | Instagram/抖音 Story、竖版海报、朋友圈竖图 |
| `4:3` | 传统横版、部分平台帖子 |
| `3:2` | 横版照片风格、博客图 |
| `2:3` | Pinterest、竖版海报、A4 比例 |
| `5:4` | 微博/部分平台方形偏横 |
| `21:9` | 超宽 banner、网站 header、封面横幅 |

**使用方式**：用户说"生成一张 9:16 的图"或"做个 16:9 文章配图"时直接按对应比例生成，不需要额外确认。默认不指定时用 `1:1`。

## 工作流

### 1. 理解内容与比例

先读用户给的内容（文章/帖子/想法）。确认：
- 内容主题是什么，适合哪种布局（读 `composition-patterns.md`）
  - 属于 YC 5大支柱？→ 读 `yc-brand-identity.md` 匹配对应场景
  - 是知识讲解/任意主题？→ 用 Knowledge Explainer 类型（类型6）
- 目标比例是什么（没说就用 `1:1`）

### 2. 先出配图策略

用户说"分析怎么配图 / shot list / 帮我想想"时，先给方案，每张写：
- 内容段落位置
- 图的主题 + 核心意思
- 布局类型
- YC 在做什么
- 建议比例
- 建议元素和标注词

文章类默认 3-6 张；单条社媒帖 1-2 张。

### 3. 单张生成

用户说"生成 / 出图 / 帮我做"时，用 `image_gen` 单张生成，不要拼在一起。

每张提示词必须包含：
- 明确比例尺寸
- YC 角色关键词（红发 + 透明眼镜 + chibi 比例）
- 暖色调 / 奶油白底
- 布局类型
- 签名道具（按场景选择）
- 中英混排标注

### 4. 检查与迭代

生成后过 `qa-checklist.md`。常见问题：
- YC 没有红发或眼镜 → 角色识别失败，重生成
- 背景是纯白而不是暖白/奶油色 → 重生成
- 没有签名道具（MacBook 心形贴纸 / 笑脸杯）→ 加回去
- 布局太空或太满 → 按类型规则调整

### 5. 保存

最终图存到：

```
assets/<content-slug>/
01-topic-name.png
02-topic-name.png
```

## 输出口径

策略输出简洁。交付包含：生成张数、每张用途、保存路径、哪张最稳。不要解释风格理论。
