# YC 个人 IP 配图

## 核心定位

用 YC 的视觉 IP 为任何内容生成配图。

内容主题不限：YC 的5大内容支柱（AI/成长/音乐/设计/生活）是最常见场景，但也可以讲数学、解释科学概念、介绍历史、配任何知识文章——只要是"YC 来给你解释"或"YC 风格呈现"就适用。

## 三种输出模式

| | Sticker 贴图模式（默认） | Simple 普通模式 | Rich 丰富模式 |
|---|---|---|---|
| **风格** | 白底水彩铅笔小贴纸，单角色 | 多场景故事流，适中文字 | 温暖手账 editorial，多面板 |
| **画风** | **水彩铅笔轻涂松手绘**（非全彩 anime，非黑线稿） | 温暖 chibi 插画 | 精致 editorial 插画 |
| **文字量** | 0-3 个中文短词（能不写就不写） | 中英混排，每步 1-2 个短标注 | 中英混排，5-10+ 个标注 |
| **布局** | 单主体，一个动作/物件，巨量留白 | 分步场景串联（3-6步） | 多卡片/面板/分区 |
| **背景** | **纯白 #FFFFFF**，无纹理 | 奶油白，轻纸感 | 奶油白，有纸感 |
| **信息密度** | 极低：一张小贴纸 | 中：一个流程讲清楚 | 高：完整知识体系 |
| **适合** | 文章配图、社媒单图、一个概念/情绪/判断 | 流程故事、学习步骤、使用方法 | IP 介绍、品牌全景、知识全面讲解 |

> ⚠️ 模型默认会跑成"精致全彩 anime + 满版拼贴"= 海报。Sticker/Simple 必须主动压制（白底、松手绘、删道具、删装饰、**带参考图**）。只有 Rich 允许满。

**如何选择（用户明确说了模式词时）**：
- "贴图/小贴图/sticker/轻量/极简/干净/少字"→ Sticker
- "简洁/simple/普通/流程/步骤"→ Simple
- "丰富/详细/rich/完整/全面/IP介绍"→ Rich

## 按用例选模式（用户没明说模式时——先看"这张图干嘛用的"）

核心心法（学 template）：**只有一种默认密度=轻(Sticker)，靠换"画的内容/结构"适配场景，不靠加密度。Rich 是例外，不是常态。怀疑时永远选更轻的那个。**

**一句话判断口诀**：
- **一件事**（一个概念/观点/判断/情绪，或一个角色在做某事）→ **Sticker**（默认）
- **多步过程**（要分 3-6 步 / 左右 before-after 才说得清）→ **Simple**
- **整页成品**（IP 自我介绍 / 品牌全景 / 一张讲完整个体系）→ **Rich**（只有这类）

> 即"一张图一个画面能说清 → Sticker；非得分步/对比 → Simple；内容本身就是一整页 → Rich"。封面/标题也是"一件事"，走 Sticker 封面版，不是 Rich。

| 用例 | 模式 | 布局要点 |
|---|---|---|
| 文章正文配图（一段一个概念/判断/隐喻） | **Sticker** | 单个 YC + 一个动作/物件，白底，一张只讲一件事 |
| 社媒单图（小红书/朋友圈一张） | **Sticker** | 同上，0-3 字 |
| **文章封面 / 标题图** | **Sticker 封面版** | 单个 YC hero（居中或一侧）+ 一行标题，白底，**不堆元素**（见 prompt-template 封面片段） |
| 概念 / 观点 / 情绪 | **Sticker** | 一个隐喻画面 |
| 流程 / 步骤 / 使用方法（3-6 步） | **Simple** | 分步场景串联，箭头连接 |
| Before/After、混乱→有序 | **Simple** | 左右对比 |
| IP 自我介绍 / 品牌全景 / 角色设定页 | **Rich** | 只有"整页本身就是成品"才用满版 |
| 一张图讲完整个知识体系 | **Rich** | 多面板 |

**Rich 只用于**：IP 介绍、品牌全景、设定页、"一张讲完整个体系"。其余先想 Sticker，讲不下再升 Simple。

### 三条防胖铁律（学 template，防止又变海报）
1. **先提认知锚点**：读完内容先挑"核心判断/转折/对比/闭环"那几个点，只给这些配图，别每段都配、别做成画册——够用就好。
2. **一张只讲一件事**：宁可多出几张轻图，也不要一张塞满。
3. **examples 只校准不照抄**：`assets/examples/` 只用来锁画风+角色，不复制具体构图；每次从当前内容重新想画面。

## 支持的图片比例

| 比例 | 用途 |
|------|------|
| `16:9` | 文章配图、博客 banner、B站封面、横版知识帖 |
| `1:1` | Instagram、小红书方图、通用社媒 |
| `9:16` | Instagram/抖音 Story、竖版海报、朋友圈竖图 |
| `4:3` | 传统横版、部分平台帖子 |
| `3:2` | 横版照片风格、博客图 |
| `2:3` | Pinterest、竖版海报、A4 比例 |
| `5:4` | 微博/部分平台方形偏横 |
| `21:9` | 超宽 banner、网站 header、封面横幅 |

**使用方式**：用户说"生成一张 9:16 的图"或"做个 16:9 文章配图"时直接按对应比例生成，不需要额外确认。默认不指定时用 `1:1`。

## 工作流

### 1. 理解内容与比例

先读用户给的内容（文章/帖子/想法）。确认：
- **用哪个模式**：按上面的口诀判断（一件事→Sticker / 多步→Simple / 整页成品→Rich），默认 Sticker
  - 属于 YC 5大支柱？→ 读 `yc-brand-identity.md` 匹配对应场景
  - 选了 Rich？→ 再读 `composition-patterns.md` 选具体布局类型（Sticker/Simple 不需要）
- 目标比例是什么（没说就用 `1:1`）

### 2. 先出配图策略

用户说"分析怎么配图 / shot list / 帮我想想"时，先给方案，每张写：
- 内容段落位置
- 图的主题 + 核心意思
- **用哪个模式**（Sticker/Simple/Rich）+（Rich 时）布局类型
- YC 在做什么
- 建议比例
- 建议元素和标注词

先提"认知锚点"——只给核心判断/转折/对比配图，够用就好，别每段都配。文章类默认 3-6 张；单条社媒帖 1-2 张。

### 3. 单张生成（必须带参考图）

用户说"生成 / 出图 / 帮我做"时，用 `image_gen` 单张生成，不要拼在一起。

**先确定模式 → 取对应参考图 → 再生图。** 纯文字 prompt 会跑成海报，每次都附风格参考图：Sticker 用 `assets/examples/sticker/`；Simple 用 `assets/examples/simple/`；Rich 用 `assets/examples/rich/`；锁角色长相加 `assets/examples/character-reference/`。第一次仍偏属正常，带参考图重生成第二次即对。

每张提示词必须包含明确比例 + YC 角色关键词（红发 + 透明眼镜 + chibi）。其余按模式：

- **Sticker（默认）**：纯白底 + 水彩铅笔轻涂松手绘 + 单角色单物件 + 0-3 中文短词 + 巨量留白。**删掉**所有签名道具、便利贴、大 logo、装饰、英文。
- **Simple**：奶油白 + 3-6 步场景流 + 每步 1-2 短标注 + 2-3 个签名道具。
- **Rich**：奶油白 + 多面板 + 中英混排 + section header + 3-4 个签名道具。

### 4. 检查与迭代

生成后过 `qa-checklist.md`。常见问题：
- YC 没有红发或眼镜 → 角色识别失败，重生成（所有模式通用）
- **Sticker**：背景不是纯白、画得太精致全彩、出现道具/便利贴/装饰/英文、信息太满 → 重生成压回贴纸感
- **Simple/Rich**：背景是纯白而不是奶油色、缺签名道具、布局太空 → 按对应模式补回
- 拿不准就回看 `assets/examples/` 里对应模式的例图对齐

### 5. 保存

最终图存到：

```
assets/<content-slug>/
01-topic-name.png
02-topic-name.png
```

## 输出口径

策略输出简洁。交付包含：生成张数、每张用途、保存路径、哪张最稳。不要解释风格理论。
